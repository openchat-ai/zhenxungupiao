# Phase 4：GUI 帧缓冲

## 运行时 Opcode（0xB0–0xB4）

| Opcode | 参数 | 含义 |
|--------|------|------|
| `B0` | `w, h` | 初始化帧缓冲 `state[1]`，宽 `state[2]`，高 `state[3]` |
| `B1` | `x, y, color` | 画像素（ARGB） |
| `B2` | `x1,y1,x2,y2,c` | 线段（发射器简化） |
| `B3` | `x,y,w,h,c` | 矩形填充 |
| `B4` | — | 呈现（WriteFile 到 stdout 或 syscall） |

## 纯 .ty 软件渲染（当前可用）

不依赖 B0–B4 时，`lib/gui.ty` + `lib/chart.ty` 用 state 槽 `[512..]` 作 48×32 帧缓冲：

| 颜色 | RGB | 用途 |
|------|-----|------|
| `#a855f7` | 紫 | **买入**信号 |
| `#2ebd85` | 绿 | **卖出**信号 |
| `#f6465d` | 红 | K 线涨 |
| `#0b0e11` | 深灰 | 背景 |

## 布局

```
┌────────────────────┐
│  K 线主图 (y=0..30) │
├────────────────────┤
│  四指标条 (y=32)    │  紫/绿点
├────────────────────┤
│  综合信号 (y=46)    │  紫买/绿卖横幅
└────────────────────┘
```

## 手机触摸（待扩展 0xB5）

| Opcode | 含义 |
|--------|------|
| `B5` | 读触摸 → `state[4]=x, state[5]=y, state[6]=pressed` |

Android/iOS 后端将 B5 映射到各自输入 API，仍零第三方库。
